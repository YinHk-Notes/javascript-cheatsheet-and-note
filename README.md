<strong>#var, let, and const – What's the Difference?</strong>
</p>
<p>

<strong>var </strong>can be re-declared and updated
</p>
<p>

<strong>let</strong> can be updated but not re-declared +<span style="text-decoration:underline;"> blocked scope</span>.
</p>
<p>

<strong>const </strong>cannot be updated or re-declared + <span style="text-decoration:underline;">blocked scope</span> + must be initialized at the time of declaration(read-only).
</p>
<ul>

<li><strong>var declarations are globally scoped or function scoped while let and const are block scoped.</strong>

<li><strong>var variables can be updated and re-declared within its scope; let variables can be updated but not re-declared; const variables can neither be updated nor re-declared.</strong>

<li><strong>They are all hoisted to the top of their scope. But while var variables are initialized with undefined, let and const variables are not initialized.</strong>

<li><strong>While var and let can be declared without being initialized, const must be initialized during declaration.</strong>

<li><strong>You can <span style="text-decoration:underline;">change/add</span> the properties of a <span style="text-decoration:underline;">constant/let object</span></strong>

<strong>#Map object: </strong>
</p>
<p>

<strong>Map is a data collection type (in a more fancy way — <em>abstract data structure type</em>), in which, data is stored in a form of pairs, which contains <span style="text-decoration:underline;">a unique key</span> and <span style="text-decoration:underline;">value</span> mapped to that key. And because of the uniqueness of each stored key, there is no duplicate pair stored.</strong>
</p>
<p>

<strong>The Map object holds key-value pairs and remembers the original insertion order of the keys. Any value (both objects and <a href="https://developer.mozilla.org/en-US/docs/Glossary/Primitive">primitive values</a>) may be used as either a key or a value.</strong>
</p>
<p>

<strong><a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object">Object</a> is similar to Map—both let you set keys to values, retrieve those values, delete keys, and detect whether something is stored at a key.</strong>
